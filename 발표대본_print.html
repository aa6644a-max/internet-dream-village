SL 01 표지 約 1분

안녕하세요.

저는 오늘 인터넷 드림마을이라는 프로젝트를 발표하겠습니다.

인터넷 드림마을은 청소년의 스마트폰 과몰입을 방지하기 위한 AI 기반 웹 상담 플랫폼입니다.

"스마트폰을 덜 써라"는 말 대신, AI가 학생과 직접 대화하며 5년 후의 자신을 이미지로 그려주고, 그 미래를 향한 동기를 스스로 키워나가도록 돕는 서비스입니다.

개발은 Claude Code를 활용한 Vibe Coding 방식으로 진행하고 있습니다. AI가 코드를 작성하고, 저는 방향 설정과 검증에 집중하는 방식입니다.

SL 03 왜 지금 해야 하는가? — 청소년 스마트폰 과몰입 현황 約 2분
↗ 포인터 → 40.1% 카드

청소년 스마트폰 과의존 위험군 비율이 40.1%입니다. 열 명 중 네 명이 위험 수준이에요. 2023년 과학기술정통부 조사 결과입니다.

↗ 포인터 → 4.2시간 카드

청소년의 일평균 스마트폰 사용 시간은 4.2시간. 성인보다 1.3배 높습니다.

↗ 포인터 → r=0.68 카드

그리고 스마트폰 사용 증가와 청소년 우울·불안 사이의 상관계수가 r = 0.68로, 통계적으로 유의미한 관계가 확인됩니다.

↗ 포인터 → 하단 문제점 목록

숫자 너머의 문제는 크게 네 가지입니다. 첫째, 기존 면대면 상담은 시간과 장소 제약이 크다. 둘째, "나중에 잘 살아야지"식의 추상적 미래 이야기로는 청소년의 행동 변화를 이끌기 어렵다. 셋째, 학생의 감정과 행동 변화 데이터가 체계적으로 축적되지 않는다. 넷째, 자해나 우울 같은 위기 신호를 빠르게 감지하고 연결하는 시스템이 없다.

이 네 가지를 동시에 해결하는 게 인터넷 드림마을의 출발점이었습니다.

SL 04 기존 방식 vs 인터넷 드림마을 約 2분
↗ 포인터 → 좌측 패널 (기존 방식)

기존 상담 방식의 한계를 보겠습니다. 면대면 상담은 접근성이 낮고, 추상적인 미래 설명은 동기 부여 효과가 제한적이며, 데이터를 체계적으로 수집하기 어렵습니다. 위기 신호도 사후에 발견되는 경우가 많습니다.

↗ 포인터 → 우측 패널 (드림마을)

인터넷 드림마을은 이 문제를 정반대로 접근합니다. 웹앱이기 때문에 스마트폰이든 PC든 언제 어디서나 접근할 수 있고, AI 이미지로 추상적인 미래를 지금 눈앞에 생생하게 보여줍니다. 마음일기와 자가진단 데이터가 자동으로 쌓이고, 위기 신호가 감지되면 관리자 선생님께 즉시 알림이 갑니다.

↗ 포인터 → 하단 핵심 차이 바

핵심 차이 한 줄 — 기존 방식은 사람이 직접 관리하고, 드림마을은 AI가 24시간 함께합니다.

SL 05 핵심 이론 배경 — EFT · CIREF · 지연 할인 극복 約 2분

이 서비스는 세 가지 과학적 이론에 기반합니다.

↗ 포인터 → EFT 카드

첫째, EFT — Episodic Future Thinking입니다. 추상적인 미래를 구체적인 에피소드로 변환하면, 뇌의 도파민 보상 회로가 건강하게 재공고화된다는 이론입니다. "나중에 잘 살아야지"가 아니라 "5년 후 나는 도서관에서 친구들과 공부하고 있다"처럼, 장소·인물·감정이 구체적일수록 행동 변화가 일어납니다.

↗ 포인터 → CIREF 카드

둘째, CIREF 프레임워크입니다. 단서 유발 회상과 일화적 미래 사고를 결합한 4단계 대화 구조입니다. 드림마을의 '미래의 나' 모듈이 바로 이 프레임워크를 그대로 구현합니다. Claude API가 학생과 대화하며 미래 에피소드를 수집하고, Gemini가 이미지로 만들어줍니다.

↗ 포인터 → 지연 할인 카드

셋째, 지연 할인 극복입니다. 사람은 먼 미래의 보상을 현재 가치보다 낮게 인식하는 경향이 있습니다. 이게 바로 "내일부터 해야지"가 반복되는 심리적 이유입니다. AI가 생성한 생생한 이미지는 이 인지 편향을 극복하게 도와주고, 즉각적인 행동 변화 동기를 만들어냅니다.

SL 06 CIREF 4단계 프로세스 約 2분
↗ 포인터 → 4단계 화살표

미래의 나 모듈은 네 단계로 진행됩니다.

1단계 시공간 묘사: "스마트폰을 내려놓은 미래의 너는 어디서 누구와 뭘 하고 있어?" — 답변이 짧으면 Claude가 "어떤 장소인지 더 구체적으로 말해줄 수 있어?"처럼 자연스럽게 보완 질문을 합니다.

2단계 감각/감정 예측: "그 장면에서 기분이 어때? 몸으로 느낀다면 어떤 느낌일까?"

3단계 정체성 탐색: "해결된 미래의 너는 네가 꿈꾸던 어떤 어른의 모습과 닮았어?"

4단계 대체 활동: "스마트폰 대신 푹 빠져서 해보고 싶은 활동은 뭐야?"

학생이 "모르겠어요"라고 짧게 답하면 소크라테스식 재질문으로 유도합니다. 2회 연속 짧은 답변이 오면 자동으로 예시 보기 버튼이 나타납니다.

↗ 포인터 → AI 분석 / 이미지 생성 카드

첫 세션에서 학생은 자신의 외모 앵커를 입력합니다. 키, 머리 스타일, 안경 착용 여부 같은 정보를 텍스트로 입력해두면, Gemini가 이미지를 생성할 때 동일 인물 일관성을 유지해줍니다.

4단계 완료 후 Gemini가 5년 후 이미지를 생성하면, 리플렉션 3문항이 순차적으로 표시됩니다. "이 미래의 나를 보고 어떤 느낌이 들어?", "지금 당장 뭔가 바꾸고 싶은 게 생겼어?", "이 미래를 위해 오늘 할 수 있는 한 가지는?" — 이 세 가지 답변이 저장됩니다. 5년 후 이후에는 "10년 후의 나도 볼까요?" 선택지가 나옵니다.

SL 07 4개 핵심 모듈 約 2분

학생 대시보드에는 네 가지 모듈이 있습니다.

↗ 포인터 → 마음일기 카드

첫째, 마음일기 (Phase 1). 매일 감정을 텍스트로 기록하면 Claude API가 감정 Valence를 추출하고 꽃 SVG로 시각화합니다. 기쁜 날은 만개한 해바라기, 슬픈 날은 고개를 숙인 꽃이 그려집니다. Phase 3에서는 Gemini가 bloom_degree, petal_count 같은 파라미터를 조절해 무한한 변형이 가능합니다. 위기 신호가 감지되면 즉시 관리자에게 알림이 발송됩니다.

↗ 포인터 → 미래의 나 카드

둘째, 미래의 나 (Phase 2 핵심). 방금 설명드린 CIREF 모듈입니다. 월 5회 기본 제공이며 관리자가 1~10회로 조정할 수 있습니다.

↗ 포인터 → 자가진단 카드

셋째, 나의 자가진단 (Phase 1). SAS 중독 척도와 PHQ-A 청소년 우울 척도로 정기 측정합니다. 여기서 주목할 기능이 있습니다. 매일 접속할 때 "오늘 스마트폰 몇 시간 썼어?"를 슬라이더로 입력받아, 스마트폰 사용량과 정서 안정도의 상관관계 그래프를 보여줍니다. 학생이 직접 "폰을 덜 쓴 날 기분이 좋았구나"를 데이터로 확인하는 메타인지가 핵심입니다. (Phase 3 고도화)

↗ 포인터 → 미로 풀기 카드

넷째, 미로 풀기 (Phase 3). 기존에 개발된 10단계 미로 프로젝트를 이식합니다. 레벨이 올라갈수록 그리드가 커지고, 플레이어 시야를 안개로 제한하는 난이도 시스템이 추가됩니다. 일일 15분 플레이 제한으로 2차 중독을 방지합니다.

SL 09 개발 로드맵 約 2분
↗ 포인터 → Phase 1 행

Phase 1 MVP — 계정 시스템, 마음일기, 자가진단, 관리자 대시보드, 위기 신호 감지, PDF 저장 기능이 포함됩니다. 이것만 완성되어도 실제 학교 현장에 투입할 수 있는 수준입니다.

↗ 포인터 → Phase 3 행

Phase 3 — 미로 게임 이식, 꽃 SVG AI 파라미터화, 스마트폰 사용량 상관관계 그래프, 수퍼바이저 대시보드, 관리자 PDF 리포트 3종. 완성형 B2B SaaS 수준입니다.

그리고 PRD에는 Phase 4도 설계되어 있습니다. Next.js를 Expo로 전환해서 웹과 Android 앱을 단일 코드베이스로 지원하고, Google Play 배포를 목표로 합니다. 이때 부모 모니터링 계정도 추가됩니다. 부모는 자녀의 자가진단 수준과 스마트폰 사용량만 확인할 수 있고, 마음일기 전문은 부모에게도 공개되지 않습니다. 학생이 솔직하게 기록하려면 그 공간이 보호받아야 하기 때문입니다.

SL 10 기술 스택 約 1분 30초
↗ 포인터 → 표 전체

프론트엔드는 Next.js 14 App Router + Tailwind CSS + shadcn/ui로 Vercel에 최적화된 반응형 웹앱을 만들고, 백엔드와 데이터베이스는 Supabase입니다. RLS 정책으로 학생 데이터를 DB 레벨에서 격리하고, Realtime으로 위기 신호 알림을 처리합니다.

↗ 포인터 → AI 엔진 행

AI 엔진은 Claude API입니다. 마음일기 감정 분석, CIREF 4단계 대화 엔진, 위기 신호 감지, 동기강화 상담 재질문 — 핵심 AI 기능 전부를 Claude가 담당합니다.

이미지 생성은 Gemini API입니다. 초기에는 fal.ai와 Replicate를 검토했지만, 텍스트 프롬프트 생성과 이미지 생성을 같은 API에서 처리할 수 있는 일관성 때문에 Gemini로 결정했습니다. 생성된 이미지는 WebP로 변환해 720px 리사이즈 후 Supabase Storage에 저장됩니다.

SL 11 마무리 約 1분

정리하겠습니다.

인터넷 드림마을은 단순한 상담 앱이 아닙니다.

EFT와 CIREF라는 과학적 이론을 바탕으로, Claude가 학생과 대화하고 Gemini가 그 이야기를 이미지로 그려주며, 매일 쌓이는 감정 데이터와 상관관계 그래프로 학생 스스로 변화를 확인합니다. 그리고 위기 순간에는 즉시 사람과 연결합니다.

"스마트폰을 끊어라"는 말 대신, "네 미래가 이렇게 근사하다"는 걸 직접 보여주는 서비스를 만들고 싶었습니다.

이상입니다. 감사합니다.

부록 B — 예상 질문 & 답변
청중 질문에 대비한 핵심 답변 요약
Q1이미지 생성 API를 fal.ai가 아니라 Gemini로 선택한 이유가 있나요?
PRD1 초기에는 fal.ai·Replicate를 검토했지만, CIREF 답변 분석(텍스트)과 이미지 생성 프롬프트 생성, 실제 이미지 생성을 같은 Gemini API에서 처리하면 연동 관리가 단순해지고 일관성이 높아진다는 판단으로 PRD2 설계 시 Gemini로 변경했습니다.
Q2이미지 일관성은 어떻게 유지하나요?
Phase 2에서는 첫 세션의 외모 앵커 텍스트(키, 머리 스타일, 안경 여부 등)를 매 프롬프트에 포함시켜 일관성을 유지합니다. Phase 4에서는 첫 세션 이미지를 참조 이미지로 사용하는 img2img 방식으로 고도화할 예정입니다.
Q3부모가 자녀의 일기를 볼 수 있나요?
볼 수 없습니다. Phase 4에 부모 모니터링 계정이 추가되지만, 마음일기 전문과 미래 이미지 갤러리는 부모에게도 공개되지 않습니다. 학생이 솔직하게 기록하려면 그 공간이 보호받아야 한다는 원칙입니다. 부모는 자가진단 수준(관심/주의/위험)과 스마트폰 사용 시간 추이, 위기 신호 알림만 받습니다.
Q4월 5회 세션 제한이 너무 적지 않나요?
관리자가 1~10회/월 범위에서 기관별로 조정할 수 있습니다. 기본값 5회는 이미지 생성 API 비용과 학생이 매 세션을 충분히 소화하는 시간을 고려한 수치입니다. 이미지 생성 실패 시에는 횟수가 차감되지 않습니다.
Q5실제 심리 상담을 대체할 수 있나요?
대체가 아닌 보조 도구입니다. 위기 신호 감지는 보조 역할이고 최종 판단은 항상 관리자(상담사)가 합니다. AI는 접근성·데이터 추적·동기 부여 역할에 집중합니다.
Q6CIREF 답변 같은 내면 데이터는 어떻게 보호하나요?
Supabase RLS로 본인과 담당 관리자만 접근 가능하도록 제한하고, CIREF 4단계 답변 원문은 암호화 저장을 권장 사항으로 명시했습니다. 이미지 생성 프롬프트 로그는 오남용 감사 목적으로 별도 보관합니다.
Q7Google Play 배포 계획은 언제인가요?
Phase 1~3 Next.js 웹 버전을 완성하고 현장 검증 후, Phase 4에서 Expo로 전환해 Android 앱으로 배포합니다. 미성년자 대상 앱이므로 Google Play 아동 정책 체크리스트(개인정보 처리방침, 행동 기반 광고 없음, 학부모 동의 메커니즘 등)를 사전에 점검 중입니다.
