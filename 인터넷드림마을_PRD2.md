## 1. 개요

### 1.1 모듈 한 줄 정의
학생이 4단계 대화를 통해 스마트폰을 내려놓은 미래 자신의 모습을 AI 이미지로 생성하고, 리플렉션을 통해 행동 변화 동기를 강화하는 EFT(Episodic Future Thinking) 핵심 모듈

### 1.2 이론 배경
- **EFT**: 추상적 미래를 구체적 에피소드로 시각화 → 뇌 도파민 보상 회로 재공고화
- **CIREF 프레임워크**: 단서 유발 회상 + 일화적 미래 사고 결합
- **지연 할인 극복**: AI 이미지로 먼 미래 보상을 현재처럼 체감

### 1.3 Phase 1 선행 조건
Phase 2 개발 전 Phase 1 (계정 시스템, 마음일기, 자가진단, 관리자 대시보드) 완료 필요

## 2. 기능 명세

### 2.1 전체 흐름

```
학생 대시보드 → "미래의 나" 메뉴 진입
  │
  ├── 신규 세션 시작 가능 여부 확인 (월 횟수 잔여 확인)
  │     └── 잔여 0회 → "이번 달 사용 완료. 다음 달에 만나요." 안내
  │
  ├── 첫 세션 여부 확인
  │     └── 첫 세션 → 외모 앵커 입력 단계
  │     └── 재방문 → 저장된 외모 앵커 확인 + 수정 옵션
  │
  ├── CIREF 4단계 대화 (마법사 + AI 재질문)
  │
  ├── 이미지 생성 (5년 후)
  │     └── 성공 → 이미지 표시 + 리플렉션 노트
  │     └── 실패 → 자동 재시도 3회 → "현재 사용량이 많아서..." 안내 + 횟수 미차감
  │
  ├── "10년 후의 나도 볼까요?" 선택
  │     ├── YES → 10년 후 이미지 생성 + 리플렉션 노트
  │     └── NO  → 5년 후만 저장 완료
  │
  └── 갤러리 저장 완료
```

---

### 2.2 외모 앵커 입력

**목적:** 이미지 생성 시 동일 인물 일관성 확보 (텍스트 기반)

**첫 세션 진입 시 표시:**
```
미래 이미지를 만들기 위해 네 모습을 알려줘!

· 키 (예: 163cm)
· 머리 스타일 (예: 단발, 긴 생머리, 짧은 머리)
· 안경 착용 여부
· 기타 특징 (선택, 예: 보조개, 곱슬머리)
```

**저장 후 동작:**
- 프로필에 저장
- 이후 세션: 저장된 정보 자동 표시 + [수정하기] 버튼 제공
- 수정 시 다음 세션부터 반영

### 2.3 CIREF 4단계 대화

**UI 방식:** 마법사(Wizard) 단계 구조 + 각 단계 내 AI 재질문

**진행 표시:** 상단 스텝 인디케이터 (1/4 → 2/4 → 3/4 → 4/4)

#### 1단계 — 시공간 묘사
```
질문: "스마트폰을 내려놓은 미래의 너는 어디서 누구와 뭘 하고 있어?"

학생 답변 입력
  └── AI 보완 질문 (답변이 짧거나 모호할 때)
      예: "어떤 장소인지 조금 더 구체적으로 말해줄 수 있어?"
          "혼자야, 아니면 누군가와 함께야?"
```

#### 2단계 — 감각/감정 예측
```
질문: "그 장면에서 기분이 어때? 스스로에 대해 어떤 생각을 하고 있어?"

학생 답변 입력
  └── AI 보완 질문
      예: "그 감정을 몸으로 느낀다면 어떤 느낌일까?"
```

#### 3단계 — 정체성 탐색
```
질문: "해결된 미래의 너는 네가 꿈꾸던 어떤 어른의 모습과 닮았어?"

학생 답변 입력
  └── AI 보완 질문
      예: "그 어른처럼 되기 위해 지금 내가 달라진 게 뭘까?"
```

#### 4단계 — 대체 활동
```
질문: "스마트폰 대신 푹 빠져서 해보고 싶은 활동은 뭐야?"

학생 답변 입력
  └── AI 보완 질문
      예: "그 활동을 하는 미래의 네 모습이 구체적으로 어떻게 보여?"
```

**저항 대응 로직:**
- "모르겠어요" / 1단어 답변 → 소크라테스식 재질문 또는 예시 제시
- 동기강화 상담(MI) 기법 적용
- 2회 연속 짧은 답변 시 → 예시 보기 버튼 표시

### 2.4 이미지 생성

#### 프롬프트 구조

```
[고정 접두어]
"한국인 청소년, 따뜻한 수채화 스타일, 1인칭 시점,
밝고 희망적인 분위기, 고해상도"

+ [외모 앵커]
"키 163cm, 단발머리, 안경 착용"

+ [AI 생성 장면 묘사]
Gemini가 CIREF 4단계 답변 분석 후 자동 생성
(장소, 인물, 행동, 감정, 대체 활동 포함)

+ [시간 변수]
5년 후: "5년 후, 만 N세"
10년 후: "10년 후, 만 N세"
```

#### 생성 API
- **사용 API:** Gemini API (이미지 생성 지원 모델 — 개발 시점 최신 버전)
- **기존 PRD1 Gemini 키 재사용**
- **출력 형식:** base64 또는 URL → WebP 변환 → Supabase Storage 업로드

#### 이미지 처리
```
생성 이미지
  → WebP 변환 + 720px 리사이즈
  → Supabase Storage 업로드
  → DB에 URL 저장
```

#### 오류 처리

**Warning:** API 실패 시 자동 재시도 3회. 전부 실패 시 아래 안내 표시:

```
"현재 사용량이 많아서 이미지 생성이 어려워요.
잠시 후 다시 시도해 주세요."
```

- 오류 시 **세션 횟수 차감 없음**
- 재시도 간격: 2초, 4초, 8초 (지수 백오프)

### 2.5 프로그레시브 리빌 UX

```
4단계 완료 → 로딩 화면 ("미래의 너를 그리고 있어... ✨")
  → 5년 후 이미지 공개
      [리플렉션 노트 — 3문항 순차 표시]
        Q1. "이 미래의 나를 보고 어떤 느낌이 들어?"
              → 답변 입력 → 다음
        Q2. "지금 당장 뭔가 바꾸고 싶은 게 생겼어?"
              → 답변 입력 → 다음
        Q3. "이 미래를 위해 오늘 할 수 있는 한 가지는?"
              → 답변 입력 → 저장
  → "10년 후의 나도 볼까요?" 선택
        ├── YES → 로딩 → 10년 후 이미지 공개 → 리플렉션 노트 3문항 반복
        └── NO  → 5년 후 이미지·리플렉션만 저장 완료
  → 갤러리로 이동 안내
```

### 2.6 갤러리

**방식:** 슬라이드쇼 (시간순, 처음 생성 → 최근)

```
◀  [          이미지          ]  ▶
         2026.03.15 — 5년 후
   "이 미래의 나를 보고 설렜어요"   ← 리플렉션 Q1 답변 표시

● ○ ○ ○ ○   (날짜 도트 인디케이터)
```

**표시 정보:**
- 이미지 (5년 후 / 10년 후 구분 뱃지)
- 생성 날짜
- 리플렉션 노트 Q1 답변 (감정 요약)
- [상세 보기] → 3문항 전체 답변 열람

## 3. 데이터 모델 추가

```sql
future_images
  - id

  - id

  - status               -- in_progress | completed | failed

  - created_at
  - completed_at

student_appearance
  - id
  - user_id              -- 1:1
  - height               -- 텍스트 (예: "163cm")
  - hair_style           -- 텍스트
  - glasses              -- boolean
  - extra_features       -- 텍스트 (선택)
  - updated_at
```

## 7. 스토리지 비용 전략

| 단계 | 조건 | 플랜 |
|---|---|---|
| MVP ~ 초기 | 학생 수 적음 | Supabase 무료 (1GB) |
| 성장기 | 학생 50명 이상 or 1년 운영 | Supabase Pro ($25/월, 100GB) |

- WebP 720px 기준 이미지 1장 ≈ 150KB
- 무료 1GB → 약 6,800장 수용
- 학생 30명 × 월 10장 × 12개월 = 3,600장/년 → 무료 플랜 충분

## 9. 개발 단계

### Phase 2 구현 순서

1. **DB 스키마 추가** — `ciref_sessions`, `student_appearance`, `future_images` 확장
2. **외모 앵커 입력 UI** — 첫 세션 진입 시 표시, 프로필 수정 연동
3. **CIREF 4단계 마법사 UI** — 단계별 질문 + AI 재질문 로직
4. **Gemini 프롬프트 생성 API** — CIREF 답변 → 이미지 프롬프트 변환
5. **Gemini 이미지 생성 API** — 호출 + 재시도 로직 + 오류 메시지
6. **이미지 처리 파이프라인** — WebP 변환 + Supabase 업로드
7. **프로그레시브 리빌 UX** — 5년후 공개 → 리플렉션 → 10년후 선택
8. **갤러리 슬라이드쇼** — 시간순 이미지 + 리플렉션 표시
9. **세션 횟수 관리** — 월 리셋, 관리자 설정 UI
10. **관리자 대시보드 확장** — Phase 2 데이터 열람 추가
11. **CIREF 결과 PDF 저장** — `usePDFExport` 훅 적용, Supabase 이미지 CORS 처리
