## 슬라이드 3 — 문제 현황 (약 2분)

`[포인터 → 40.1% 카드]`

청소년 스마트폰 과의존 위험군 비율이 **40.1%**입니다.

열 명 중 네 명이 위험 수준이에요. 2023년 과학기술정통부 조사 결과입니다.

`[포인터 → 4.2시간 카드]`

청소년의 일평균 스마트폰 사용 시간은 **4.2시간**. 성인보다 1.3배 높습니다.

`[포인터 → r=0.68 카드]`

그리고 스마트폰 사용 증가와 청소년 우울·불안 사이의 상관계수가 **0.68**로, 통계적으로 유의미한 관계가 확인됩니다.

`[포인터 → 하단 문제점 목록]`

숫자 너머의 문제는 크게 네 가지입니다.

첫째, 기존 면대면 상담은 시간과 장소 제약이 크다.  
둘째, "나중에 잘 살아야지"식의 추상적 미래 이야기로는 청소년의 행동 변화를 이끌기 어렵다.  
셋째, 학생의 감정과 행동 변화 데이터가 체계적으로 축적되지 않는다.  
넷째, 자해나 우울 같은 위기 신호를 빠르게 감지하고 연결하는 시스템이 없다.

이 네 가지를 동시에 해결하는 게 인터넷 드림마을의 출발점이었습니다.

## 슬라이드 4 — 기존 방식 vs 드림마을 (약 2분)

`[포인터 → 좌측 패널]`

기존 상담 방식의 한계를 보겠습니다.

면대면 상담은 접근성이 낮고, 추상적인 미래 설명은 동기 부여 효과가 제한적이며, 데이터를 체계적으로 수집하기 어렵습니다. 위기 신호도 사후에 발견되는 경우가 많습니다.

`[포인터 → 우측 패널]`

인터넷 드림마을은 이 문제를 정반대로 접근합니다.

웹앱이기 때문에 스마트폰이든 PC든 언제 어디서나 접근할 수 있고, AI 이미지로 추상적인 미래를 지금 눈앞에 생생하게 보여줍니다. 마음일기와 자가진단 데이터가 자동으로 쌓이고, 위기 신호가 감지되면 관리자 선생님께 즉시 알림이 갑니다.

`[포인터 → 하단 바]`

핵심 차이 한 줄 — **기존 방식은 사람이 직접 관리하고, 드림마을은 AI가 24시간 함께합니다.**

## 슬라이드 5 — 핵심 이론 배경 (약 2분)

이 서비스는 세 가지 과학적 이론에 기반합니다.

`[포인터 → EFT 카드]`

첫째, **EFT — Episodic Future Thinking**입니다.

추상적인 미래를 구체적인 에피소드로 변환하면, 뇌의 도파민 보상 회로가 건강하게 재공고화된다는 이론입니다. "나중에 잘 살아야지"가 아니라 "5년 후 나는 도서관에서 친구들과 공부하고 있다"처럼, 장소·인물·감정이 구체적일수록 행동 변화가 일어납니다.

`[포인터 → CIREF 카드]`

둘째, **CIREF 프레임워크**입니다.

단서 유발 회상과 일화적 미래 사고를 결합한 4단계 대화 구조입니다. 드림마을의 '미래의 나' 모듈이 바로 이 프레임워크를 그대로 구현합니다. Claude API가 학생과 대화하며 미래 에피소드를 수집하고, Gemini가 이미지로 만들어줍니다.

`[포인터 → 지연 할인 카드]`

셋째, **지연 할인 극복**입니다.

사람은 먼 미래의 보상을 현재 가치보다 낮게 인식하는 경향이 있습니다. 이게 바로 "내일부터 해야지"가 반복되는 심리적 이유입니다. AI가 생성한 생생한 이미지는 이 인지 편향을 극복하게 도와주고, 즉각적인 행동 변화 동기를 만들어냅니다.

`[슬라이드 전환]`

## 슬라이드 6 — CIREF 4단계 프로세스 (약 2분)

`[포인터 → 4단계 화살표]`

미래의 나 모듈은 네 단계로 진행됩니다.

1단계 **시공간 묘사**: "스마트폰을 내려놓은 미래의 너는 어디서 누구와 뭘 하고 있어?"

2단계 **감각/감정 예측**: "그 장면에서 기분이 어때? 몸으로 느낀다면 어떤 느낌일까?"

3단계 **정체성 탐색**: "해결된 미래의 너는 네가 꿈꾸던 어떤 어른의 모습과 닮았어?"

4단계 **대체 활동**: "스마트폰 대신 푹 빠져서 해보고 싶은 활동은 뭐야?"

`[포인터 → AI 분석 카드]`

학생이 "모르겠어요"라고 짧게 답하면 Claude가 소크라테스식 재질문으로 유도합니다. 2회 연속 짧은 답변이 오면 자동으로 예시 보기 버튼이 나타나고, 동기강화 상담 기법을 적용해 자연스럽게 이야기를 이끌어냅니다.

`[포인터 → 이미지 생성 카드]`

그리고 중요한 게 하나 있습니다. 첫 세션에서 학생은 자신의 **외모 앵커**를 입력합니다. 키, 머리 스타일, 안경 착용 여부 같은 정보를 텍스트로 입력해두면, Gemini가 이미지를 생성할 때 동일 인물 일관성을 유지해줍니다.

4단계 답변이 완료되면 Gemini가 **5년 후 이미지**를 생성하고, 이어서 이미지를 보며 세 가지 리플렉션 질문에 답하게 됩니다. "이 미래의 나를 보고 어떤 느낌이 들어?", "지금 당장 뭔가 바꾸고 싶은 게 생겼어?", 마지막으로 "이 미래를 위해 오늘 할 수 있는 한 가지는?" — 이 세 가지 답변이 저장됩니다.

5년 후 이미지 이후에는 **10년 후의 나도 볼까요?** 라는 선택지가 나옵니다. YES를 선택하면 같은 방식으로 10년 후 이미지를 생성하고 리플렉션을 합니다.

`[슬라이드 전환]`

## 슬라이드 7 — 4개 핵심 모듈 (약 2분)

학생 대시보드에는 네 가지 모듈이 있습니다.

`[포인터 → 마음일기]`

첫째, **마음일기**. 매일 감정을 텍스트로 기록하면 Claude API가 감정 Valence를 추출하고 꽃 SVG로 시각화합니다. 기쁜 날은 만개한 해바라기, 슬픈 날은 고개를 숙인 꽃이 그려지는 식입니다. 자해나 우울 키워드가 감지되면 관리자에게 즉시 알림이 발송됩니다. Phase 1 구현입니다.

`[포인터 → 미래의 나]`

둘째, **미래의 나** — 방금 설명드린 CIREF 핵심 모듈입니다. 월 5회 기본 제공이며 관리자가 1~10회로 조정할 수 있습니다. Phase 2 핵심 기능입니다.

`[포인터 → 자가진단]`

셋째, **나의 자가진단**. SAS 스마트폰 중독 척도와 PHQ-A 청소년 우울 척도를 기반으로 정기 측정합니다. 여기서 주목할 기능이 하나 있는데요 — 매일 접속할 때 "오늘 스마트폰 몇 시간 썼어?"를 슬라이더로 간단히 입력받아서, **스마트폰 사용량 감소와 정서 안정도의 상관관계 그래프**를 보여줍니다. 학생이 직접 "폰을 덜 쓴 날 기분이 좋았구나"를 데이터로 확인하는 메타인지 효과가 핵심입니다. Phase 3 고도화입니다.

`[포인터 → 미로 풀기]`

넷째, **미로 풀기**. 기존에 개발된 10단계 미로 프로젝트를 이식합니다. 레벨이 올라갈수록 그리드가 커지고, 플레이어 시야를 안개로 제한하는 난이도 시스템이 추가됩니다. 포인트와 뱃지 시스템이 있고, 중독 방지를 위해 일일 15분 플레이 제한이 걸려 있습니다. Phase 3 예정입니다.

`[슬라이드 전환]`

## 슬라이드 8 — 계정 구조 (약 1분 30초)

`[포인터 → 수퍼바이저 행]`

계정은 3계층 구조입니다.

최상위 **수퍼바이저**는 모든 기관 관리자 계정을 생성하고 구독 만료일을 수동으로 연장합니다. 결제 시스템 없이 구두 재계약 방식으로 운영해 초기 운영 부담을 최소화했습니다. 만료 D-30, D-7에 자동 이메일 알림이 발송됩니다.

`[포인터 → 관리자 행]`

중간의 **관리자**, 즉 기관 담당 선생님은 산하 학생 전체의 마음일기, 자가진단, 미래 이미지 갤러리를 열람하고, 위기 신호가 감지되면 즉시 알림을 받습니다. 학생이 7일 이상 미접속해도 알림이 갑니다.

`[포인터 → 학생 행]`

하단의 **학생**은 본인 데이터만 접근 가능합니다. 다른 학생의 데이터는 Supabase RLS, 즉 Row Level Security로 데이터베이스 레벨에서 완전히 차단됩니다. 미성년자 데이터를 다루는 만큼 보안이 설계의 핵심입니다.

`[슬라이드 전환]`

## 슬라이드 9 — 개발 로드맵 (약 2분)

`[포인터 → Phase 1 행]`

개발은 단계적으로 진행됩니다.

**Phase 1 MVP** — 계정 시스템, 마음일기, 자가진단, 관리자 대시보드, 위기 신호 감지, PDF 저장 기능이 포함됩니다. 이것만 완성되어도 실제 학교 현장에 투입할 수 있는 수준입니다.

`[포인터 → Phase 2 행]`

**Phase 2** — EFT의 핵심인 미래의 나 모듈을 구현합니다. 외모 앵커, CIREF 4단계 대화, Gemini 이미지 생성, 갤러리 슬라이드쇼가 포함됩니다. CIREF 답변은 미성년자 내면 데이터이므로 암호화 저장합니다.

`[포인터 → Phase 3 행]`

**Phase 3** — 미로 게임 이식, 꽃 SVG AI 파라미터화, 스마트폰 사용량 상관관계 그래프, 수퍼바이저 대시보드, 그리고 관리자 PDF 리포트 전체가 완성됩니다. 완성형 B2B SaaS 수준입니다.

그리고 PRD4에 **Phase 4**도 설계되어 있습니다. Next.js를 Expo로 전환해서 웹과 Android 앱을 단일 코드베이스로 지원하고, Google Play 배포를 목표로 합니다. 이때 **부모 모니터링 계정**도 추가됩니다. 부모는 자녀의 자가진단 수준과 스마트폰 사용량만 확인할 수 있고 — 마음일기 전문은 프라이버시 보호를 위해 부모에게도 공개되지 않습니다. 솔직한 기록을 방해하지 않기 위해서입니다.

`[슬라이드 전환]`

## 슬라이드 10 — 기술 스택 (약 1분 30초)

`[포인터 → 표 전체]`

기술 스택을 간단히 말씀드리겠습니다.

프론트엔드는 **Next.js 14 App Router** + Tailwind CSS + shadcn/ui로 Vercel에 최적화된 반응형 웹앱을 만들고,

백엔드와 데이터베이스는 **Supabase**입니다. RLS 정책으로 학생 데이터를 DB 레벨에서 격리하고, Realtime 기능으로 위기 신호 알림을 처리합니다.

`[포인터 → AI 엔진 행]`

AI 엔진은 **Claude API**입니다. 마음일기 감정 분석, CIREF 4단계 대화 엔진, 위기 신호 감지, 동기강화 상담 재질문 — 핵심 AI 기능 전부를 Claude가 담당합니다.

이미지 생성은 **Gemini API**입니다. Phase 2 설계 과정에서 처음에는 fal.ai와 Replicate를 검토했지만, 텍스트 기반 프롬프트 생성과 이미지 생성을 같은 API에서 처리할 수 있는 일관성 때문에 Gemini로 최종 결정했습니다. 생성된 이미지는 WebP로 변환해 720px로 리사이즈한 후 Supabase Storage에 저장됩니다.

배포는 **Vercel**, 이미지 처리는 **Sharp** 라이브러리, PDF 저장은 **html2canvas + jsPDF**를 사용합니다.

`[슬라이드 전환]`

## 슬라이드 11 — 마무리 (약 1분)

정리하겠습니다.

인터넷 드림마을은 단순한 상담 앱이 아닙니다.

EFT와 CIREF라는 과학적 이론을 바탕으로, Claude가 학생과 대화하고 Gemini가 그 이야기를 이미지로 그려주며, 매일 쌓이는 감정 데이터와 상관관계 그래프로 학생 스스로 변화를 확인합니다. 그리고 위기 순간에는 즉시 사람과 연결합니다.

"스마트폰을 끊어라"는 말 대신, "네 미래가 이렇게 근사하다"는 걸 직접 보여주는 서비스를 만들고 싶었습니다.

이상입니다. 감사합니다.

**[마음일기 시연]**

왼쪽 사이드바에서 마음일기를 클릭합니다. `[클릭]`

오늘 기분을 선택하고 `[감정 이모지 클릭]`, 일기를 작성합니다. `[예시 텍스트 입력]`

"AI 감정 분석하기" 버튼을 누르면 `[버튼 클릭]` — Claude API가 감정 Valence를 추출하는 동안 로딩이 표시됩니다.

결과를 보시면 `[결과 화면]` 감정에 따라 꽃이 다르게 그려지고, Valence 점수와 AI 코멘트가 함께 나타납니다.

---

**[미래의 나 CIREF 시연]**

미래의 나로 이동합니다. `[사이드바 클릭]`

AI가 먼저 첫 번째 질문을 던집니다. `[첫 번째 AI 말풍선]`

아래 예시 칩을 클릭하거나 직접 입력할 수 있습니다. `[예시 칩 클릭 → 전송]`

`[2~3단계 빠르게 진행]`

4단계 대화가 끝나면 Gemini가 이미지를 생성합니다. `[로딩 → 결과]`

학생이 직접 말한 이야기가 시각적 이미지가 됩니다. 이어서 리플렉션 노트를 작성하고 저장합니다.

**[자가진단 시연]**

결과 화면에서 수준 판정과 영역별 점수 차트, 그리고 AI 권고사항이 제공됩니다. `[결과 화면]`

PDF 저장 버튼은 Phase 1 구현 예정입니다.

## 예상 질문 & 답변

**Q. 이미지 생성 API를 fal.ai가 아니라 Gemini로 선택한 이유가 있나요?**
> PRD1 초기에는 fal.ai와 Replicate를 검토했습니다. PRD2 설계 과정에서 CIREF 답변 분석(텍스트)과 이미지 생성 프롬프트 생성, 그리고 실제 이미지 생성을 같은 Gemini API에서 처리하면 API 연동 관리가 단순해지고 일관성이 높아진다는 판단으로 Gemini로 변경했습니다.

**Q. 이미지 일관성은 어떻게 유지하나요?**
> Phase 2에서는 첫 세션에 학생이 입력한 외모 앵커 텍스트(키, 머리 스타일, 안경 여부 등)를 매 이미지 프롬프트에 포함시켜 일관성을 유지합니다. Phase 4에서는 첫 세션 생성 이미지를 참조 이미지로 사용하는 img2img 방식으로 고도화할 예정입니다.

**Q. 부모가 자녀의 일기를 볼 수 있나요?**
> 볼 수 없습니다. Phase 4에 부모 모니터링 계정이 추가되지만, 마음일기 전문과 미래 이미지 갤러리는 부모에게도 공개되지 않습니다. 학생이 솔직하게 감정을 기록하려면 그 공간이 보호받아야 한다고 판단했습니다. 부모는 자가진단 수준(관심/주의/위험)과 스마트폰 사용 시간 추이, 위기 신호 알림만 받을 수 있습니다.

**Q. 월 5회 세션 제한이 너무 적지 않나요?**
> 관리자가 1~10회 범위에서 기관별로 조정할 수 있습니다. 기본값 5회는 이미지 생성 API 비용과 학생이 매 세션을 충분히 소화하는 시간을 고려한 기본값입니다. 이미지 생성 실패 시에는 횟수가 차감되지 않습니다.

**Q. 실제 심리 상담을 대체할 수 있나요?**
> 대체가 아닌 보조 도구입니다. 위기 신호 감지는 보조 역할이고, 최종 판단은 항상 관리자(상담사)가 합니다. AI는 접근성과 데이터 추적, 동기 부여를 높이는 역할에 집중합니다.

**Q. CIREF 답변 같은 민감한 내면 데이터는 어떻게 보호하나요?**
> Supabase RLS로 본인과 담당 관리자만 접근 가능하도록 제한하고, CIREF 4단계 답변 원문은 암호화 저장을 권장 사항으로 명시했습니다. 이미지 생성에 사용된 프롬프트 로그는 오남용 감사 목적으로 별도 보관합니다.

**Q. Google Play 배포 계획은 언제인가요?**
> Phase 1~3 Next.js 웹 버전을 완성하고 실제 학교 현장에서 검증한 뒤, Phase 4에서 Expo로 전환해 Android 앱으로 배포할 계획입니다. 미성년자 대상 앱이므로 Google Play의 아동 정책 체크리스트(개인정보 처리방침, 행동 기반 광고 없음, 학부모 동의 메커니즘 등)를 사전에 점검하고 있습니다.
