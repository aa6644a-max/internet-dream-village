# 인터넷 드림마을 (Internet Dream Village)
## 스마트폰 과몰입 방지 AI 상담 웹앱 — Product Requirements Document (PRD)
**버전:** v1.0  
**작성일:** 2026-05-20  
**개발 방식:** Claude Code (Vibe Coding)

---

## 1. 프로젝트 개요

### 1.1 서비스 한 줄 정의
청소년의 스마트폰 과몰입을 줄이기 위해 **AI 기반 미래 시각화(EFT)** 와 **정서 기록** 을 결합한 웹 기반 상담 플랫폼

### 1.2 핵심 이론 배경
- **EFT (Episodic Future Thinking)**: 추상적 미래를 구체적 에피소드로 변환하여 뇌의 도파민 보상 회로를 건강하게 재공고화
- **CIREF 프레임워크**: 단서 유발 회상 + 일화적 미래 사고 결합 재공고화
- **지연 할인(Delay Discounting) 극복**: AI 이미지 생성으로 먼 미래 보상을 현재처럼 체감하게 유도

### 1.3 대상 사용자
| 역할 | 설명 |
|---|---|
| 학생 (청소년) | 스마트폰 과몰입 방지 프로그램 참여자 |
| 관리자 | 기관 담당자 (교사, 상담사 등) — 전체 학생 현황 관찰 |
| 수퍼바이저 | 시스템 운영자 — 관리자 계정 및 구독 현황 관리 |

## 2. 계정 구조 및 권한

### 2.1 3-Tier 계정 체계

```
수퍼바이저 계정 (1개)
    └── 관리자 계정 (N개, 기관별)
            └── 학생 계정 (N개, 관리자 산하)
```

### 2.2 수퍼바이저 계정
- 모든 관리자 계정 생성·삭제·정지
- 각 관리자의 **최초 가입일**, **마지막 로그인일**, **누적 사용 기간** 확인
- 구독 만료 임박 관리자에게 알림 발송
- 구독 갱신은 **유료 결제 없이 구두 재계약 방식** → 수퍼바이저가 수동으로 만료일 연장
- 전체 서비스 통계 대시보드 열람

### 2.3 관리자 계정
- 산하 학생 계정 일괄 생성·관리
- 모든 학생의 **마음일기**, **자가진단 점수**, **미래 이미지 갤러리**, **위기 신호** 열람
- 학생 개인 프로필 및 메모 작성
- 위기 신호(자해·우울 키워드) 감지 시 즉시 알림 수신
- **학생은 다른 학생의 데이터를 볼 수 없음**

### 2.4 학생 계정
- 본인 데이터만 접근 가능 (타 학생 데이터 완전 차단)
- 관리자가 부여한 ID/PW로 최초 로그인
- 사용 기간: 관리자 계정의 구독 기간에 종속 (기본 1년)

## 3. 기능 명세

### 3.1 학생 대시보드 — 핵심 4개 모듈

---

#### MODULE 1. 마음일기 (감정 기록 + AI 꽃 표현)

**목적:** 매일 감정을 기록하고, AI가 그 감정을 꽃 이미지로 시각화

**기능 흐름:**
1. 학생이 오늘의 감정/하루 일과를 자유 텍스트로 작성
2. AI(Claude API)가 텍스트를 분석하여 감정 Valence 추출
3. 감정 상태를 꽃의 종류·색상·형태로 표현한 이미지 또는 SVG 생성
   - 예) 기쁨 → 노란 해바라기, 불안 → 닫힌 꽃봉오리, 평온 → 라벤더
4. 감정 꽃 갤러리: 날짜별로 쌓여가는 꽃들을 달력/타임라인으로 시각화
5. 스마트폰 사용 시간 감소량과 감정 변화 그래프를 함께 표시

**AI 처리 요건:**
- 입력: 학생 일기 텍스트
- 출력: 감정 키워드 3개 + 꽃 SVG 파라미터 (종류, 색상, 개화도)
- 위기 신호 감지: 자해·우울 키워드 검출 시 관리자 알림 트리거

---

#### MODULE 2. 미로 풀기 (마음 안정 게임)

**목적:** 가벼운 두뇌 활동으로 스마트폰 대신 할 수 있는 디지털 대안 활동 제공

**기능 흐름:**
1. 매일 새로운 미로 1개 제공 (난이도: 쉬움/보통/어려움 선택)
2. 미로 클리어 시 포인트 적립 → 학생 프로필 내 뱃지 시스템
3. 클리어 기록은 관리자 대시보드에서 참여도 지표로 활용

**확장 가능 게임 아이디어 (추후 단계):**
- 단어 퍼즐 (감정 관련 낱말 찾기)
- 색칠하기 (감정 표현 도구)
- 기억력 카드 게임

**설계 원칙:** 디지털 게임 자체가 2차 중독이 되지 않도록 일일 플레이 시간 상한 설정 (예: 15분)

---

#### MODULE 3. 미래의 나 — AI 자화상 생성 (핵심 EFT 모듈)

**목적:** 5년 후·10년 후의 긍정적 미래를 AI 이미지로 생성하여 행동 변화 동기 부여

**기능 흐름 (4단계 CIREF):**

| 단계 | 내용 |
|---|---|
| 1단계. 질문 수집 | 4개 카테고리 질문에 대화형으로 답변 수집 |
| 2단계. 분석 | Claude API: 장소·인물·사물 Entity 추출 + 감정 키워드 분석 |
| 3단계. 이미지 생성 | 추출 데이터 → 이미지 생성 API (1인칭 시점, 고해상도) |
| 4단계. 재공고화 | 생성된 이미지 확인 후 신체적·심리적 반응 질문 + 리플렉션 노트 저장 |

**4개 질문 카테고리:**
1. **시공간 묘사**: "스마트폰을 내려놓은 미래의 너는 어디서 누구와 뭘 하고 있어?"
2. **감각/감정 예측**: "그 장면에서 기분이 어때? 스스로에 대해 어떤 생각을 하고 있어?"
3. **정체성 탐색**: "해결된 미래의 너는 네가 꿈꾸던 어떤 어른의 모습과 닮았어?"
4. **대체 활동**: "스마트폰 대신 푹 빠져서 해보고 싶은 활동은 뭐야?"

**저항 대응 로직:**
- "모르겠어요" 답변 시 → 소크라테스식 재질문 또는 예시 제시로 유도
- 동기강화 상담(MI) 기법 적용

**이미지 출력:**
- 5년 후 / 10년 후 각각 생성
- 생성된 이미지는 개인 갤러리에 날짜별 저장
- 이미지 변천사: 처음 생성 ~ 최근까지 슬라이드로 확인 가능

---

#### MODULE 4. 나의 중독 자가진단

**목적:** 스마트폰 의존도와 정서 상태를 정기 측정하여 변화 추이 시각화

**측정 항목:**
- 스마트폰 중독 척도 (SAS 기반 간소화 버전, 10문항 내외)
- 우울/불안 척도 (PHQ-A 청소년 버전 참고, 간소화)

**출력:**
- 점수 결과 + 수준 설명 (관심/주의/위험)
- **스마트폰 사용 감소량 × 정서 안정도 상관관계** 그래프 (메타인지 자극)
- 이전 검사와 비교한 변화 표시

**검사 주기:** 최초 1회 필수 + 권장 2주 1회 (강제 아님)

---

### 3.2 관리자 대시보드

**학생 목록 뷰:**
- 전체 학생 리스트 (이름, 마지막 접속일, 자가진단 수준, 위기 신호 여부)
- 학생 개별 클릭 시 상세 페이지 이동

**학생 개별 상세 페이지:**
- 마음일기 전체 열람
- 자가진단 점수 추이 그래프
- 미래 이미지 갤러리 (생성 변천사)
- 언어적 변화 지표: 미래 시제·행동 지향 동사 사용 빈도 추이
- 위기 신호 이력 (감지 일시, 키워드, 처리 여부)
- 관리자 메모 기록

**알림 시스템:**
- 위기 신호 감지 시 즉시 알림 (웹 푸시 또는 이메일)
- 학생이 7일 이상 미접속 시 알림

**KPI 요약 카드:**
- 전체 학생 수 / 이번 주 활성 사용자 / 평균 자가진단 개선율

---

### 3.3 수퍼바이저 대시보드

- 관리자 계정 목록 (기관명, 가입일, 마지막 로그인, 구독 만료일, 학생 수)
- 구독 만료일 수동 연장 기능
- 만료 D-30 / D-7 자동 알림
- 전체 서비스 사용 통계 (월별 활성 기관 수, 학생 수 추이)

## 4. 기술 스택 권고 (Claude Code 개발 기준)

### 4.1 프론트엔드
- **Framework:** Next.js 14 (App Router)
- **Styling:** Tailwind CSS
- **UI Components:** shadcn/ui
- **차트:** Recharts
- **상태관리:** Zustand 또는 React Context

### 4.2 백엔드 / DB
- **BaaS:** Supabase (PostgreSQL + Auth + Storage + Realtime)
  - 계정별 Row Level Security(RLS)로 데이터 격리 구현
  - 학생 데이터 타 학생 접근 차단을 DB 레벨에서 보장
- **API Routes:** Next.js API Routes 또는 Supabase Edge Functions

### 4.3 AI 연동
- **Claude API (Anthropic):** 마음일기 감정 분석, EFT 대화 엔진, 위기 신호 감지
  - 모델: `claude-sonnet-4-20250514`
- **이미지 생성 API:** Replicate (Stable Diffusion) 또는 fal.ai
  - 미래 자화상 이미지 생성용
  - 1인칭 시점, 사실적 스타일 프롬프트 엔지니어링 필요

### 4.4 인프라
- **배포:** Vercel (Next.js 최적화)
- **도메인:** 별도 구매 또는 Vercel 서브도메인 사용
- **파일 스토리지:** Supabase Storage (생성 이미지 보관)

## 5. 데이터 모델 (핵심 테이블)

```
users
  - id, email, role (student | admin | supervisor)
  - created_at, last_login_at
  - admin_id (학생의 경우 소속 관리자 FK)
  - subscription_expires_at (관리자 계정용)

diary_entries
  - id, user_id, content (텍스트), emotion_keywords[], flower_svg
  - sentiment_valence (float), created_at

self_diagnosis
  - id, user_id, score, level (safe|caution|danger)
  - responses (JSON), created_at

future_images
  - id, user_id, year_offset (5|10), image_url
  - reflection_note, entity_data (JSON), created_at

crisis_signals
  - id, user_id, detected_keyword, diary_entry_id
  - is_handled, handled_by, created_at

maze_sessions
  - id, user_id, difficulty, completed, time_taken_sec, created_at
```

## 6. 보안 및 개인정보

- **데이터 격리:** Supabase RLS 정책으로 학생은 본인 데이터만 조회 가능
- **암호화:** 민감 텍스트(일기) 저장 시 암호화 권장
- **위기 신호 처리:** 자동 감지는 보조 도구 — 최종 판단은 관리자(상담사)
- **미성년자 데이터:** 개인정보보호법 및 아동·청소년 관련 규정 준수
- **이미지 보관:** 생성된 미래 이미지는 학생 본인과 담당 관리자만 열람

## 7. 개발 단계 (Phase)

### Phase 1 — MVP (핵심 기능)
- [ ] 계정 시스템 (3-tier 로그인/권한)
- [ ] 마음일기 + AI 감정 분석
- [ ] 자가진단 (중독 척도)
- [ ] 관리자 기본 대시보드 (학생 목록 + 개별 열람)
- [ ] 위기 신호 감지 + 알림

### Phase 2 — 핵심 EFT
- [ ] 미래의 나 AI 자화상 생성 (4단계 대화 + 이미지 API)
- [ ] 이미지 갤러리 (변천사)
- [ ] 재공고화 리플렉션 노트

### Phase 3 — 게임 & 고도화
- [ ] 미로 풀기 게임
- [ ] 꽃 감정 SVG 고도화 (AI 생성)
- [ ] 스마트폰 사용량 × 정서 상관관계 그래프
- [ ] 수퍼바이저 구독 관리 대시보드

### Phase 4 — 추후 고도화
- [ ] 개인 맞춤형 아바타 고정 기술 (Consistent Character Generation)
- [ ] 모바일 앱 전환 (React Native 또는 PWA)
- [ ] 부모 모니터링 계정 추가

## 8. 비기능 요건

| 항목 | 요건 |
|---|---|
| 반응형 | 모바일 우선 디자인 (청소년 스마트폰 사용 환경) |
| 성능 | 페이지 로드 3초 이내 (이미지 생성 제외) |

## 9. 디자인 방향

- **전반적 톤:** 따뜻하고 안전한 느낌 (밝은 파스텔 계열, 부드러운 곡선 UI)
- **학생 페이지:** 친근하고 게임적 요소 (아이콘, 애니메이션)
- **관리자 페이지:** 데이터 중심, 클린한 대시보드 스타일
- **꽃 감정 표현:** 부드러운 수채화 느낌 SVG 또는 일러스트 스타일
